About Authable

With Authable you can choose the 2nd factor from a list of available options.

  • Time Based OTP (Ready)
  • QR Code scan (Coming soon)
  • Verification Code Select (Coming soon)
  • Regular OTP (Coming soon)
  • Security questions (Coming soon)
  • App Based verification

Our Philosophy

Why we think Authable makes sense...

  • 01 2nd Factor should be a Seperate System?

    If you core system's authentication was compromised assuming an internal team. The separation of the 2nd factor's independant solution protects you.

  • 02 Don't Re-implement?

    If you have more than one application that needs 2FA, you shouldn't be building 2FA in each system. Bad Practice.

  • 03 Leave it to the experts

    Best practices are important. Not just a functioning module. Choose a solution which is widely adopted.

  • 04 Bad User Experience

    Choosing a single 2FA for multiple faces to your end users is the ideal way. However, Authable allows you both options. :p
